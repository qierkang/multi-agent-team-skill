# 长期线程创建与复用策略

## 评分

| 条件 | 分数 |
|---|---:|
| 预计超过一天 | +2 |
| 至少三个任务包 | +2 |
| 独立业务或目录边界 | +2 |
| 需要持续维护 | +2 |
| 独立测试或发布 | +1 |
| 需要长期保存决策 | +1 |
| 可与其他领域并行 | +1 |

- 0-3：主线程处理。
- 4-6：当前任务内使用一次性子智能体。
- 7 以上：创建或复用长期线程。

## 生产约束

- 活跃长期线程默认不超过 5，总子智能体并发不超过 6，同时写代码不超过 2。
- `domain_key = 项目标识::领域标识`，相同领域键只能有一个活跃线程。
- 创建前先检查领域键、活跃数量、文件所有权和现有线程健康状态。
- 长期线程只负责一个稳定领域；跨领域依赖通过主线程协调，禁止线程间形成环形派发。
