# 迁移规则

## 状态处置

| 当前状态 | 默认动作 |
|---|---|
| 已完成且证据齐全 | 收口摘要后归档 |
| 已完成但证据不全 | 补证据，不立即归档 |
| 进行中且无冲突 | 允许原任务完成当前最小闭环 |
| 连续两次同因失败 | 冻结现场，用 debugger/architect 新实例升级 |
| 等待用户输入 | 保留，不代替用户回答 |
| 失联或长时间无进展 | 保存最后快照，标记待人工决定 |
| 文件所有权重叠 | 暂停后启动者，先分离路径 |

## 角色映射

标题与历史编号只能提供弱信号。最终映射依据：任务目标、实际修改路径、工具权限、验证责任和是否需要独立上下文。

## reviewer 特例

长期挂起的 reviewer 必须收口，不继续承担下一轮审查。每次审查都创建新实例，只接收最终 diff、验收标准、测试输出和必要边界。

## 执行边界

审计阶段禁止：自动归档任务、清理 worktree、reset 用户修改、覆盖 TOML、合并分支、推送或发布。上述动作必须在报告确认后逐项执行。
